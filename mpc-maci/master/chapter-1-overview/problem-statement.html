Problem statement

Problem statement

Stub

MACI, minimal anti-collusion infrastructure is a voting system, suggested here: https://ethresear.ch/t/minimal-anti-collusion-infrastructure/5413 by Vitalik Buterin, and implemented here: https://github.com/privacy-scaling-explorations/maci.

We are aiming to improve it by swapping a singular server for an MPC in such a way that user privacy could be broken only by collusion of parties in MPC.

Other desired properties - non-coercibility and auditability should still hold.

We mainly consider some simmilar settings:

  1. Third party is trying to force user into voting as they please.
  2. Third party is trying to waste user's voice.
  3. Third party is trying to buy user's voice.
  4. User themselves is trying to sell their voice.

We assume that a third party can force user into some sort of a MPC protocol.

Some examples of possible attacks:

  1. Third pary forses user to engage in a 2-party protocol in which user will sign a transaction with unknown content with their private key.
  2. If third party can prove that there are at least kkk transactions from the user published on a blockchain, they can force user to provide third party with contents of kkk different transactions, then use 1.1.1. and check if there are such blocks on blockchain.
