Unbounded fan-in OR

Unbounded fan-in OR

Source paper suggests some elaborate argument, but actually we can just negate every bit (xi→1−xix_i \rightarrow 1-x_ixi​→1−xi​), multiply them and negate the answer back, so we don't really see what's the point. It is also much faster.

Prefix-or is computed similarly, using running product construction.
